# 通用提示词框架生成器

> 不绑定平台、品类、风格的元框架。  
> 骨架定义结构，变量填充内容——任意创意需求均可模板化。

## 一句话

**骨架 + 变量 = 最终提示词。** 换材料不换模具，每个输出结构一致、质量稳定、变量可追溯。

## 解决什么问题

直接写 prompt 的痛点：
- 每次从零想结构，质量忽高忽低
- 漏掉关键维度，输出不完整
- 风格描述模糊（「日系」「高级感」「科技感」），AI 无法执行
- 中英文来回翻译，效率低

这个框架用**工业化的三步法**解决：骨架（一次定义）→ 变量填充（每次填内容）→ 输出（中英双语）。

## 快速开始

对 Claude 说：

> 帮我生成一张咖啡液抖音封面提示词，日系户外风，主文案「3秒速溶」副文案「冷热双泡」

框架自动走完 5 阶段：确定骨架 → 变量填充 → 锚点拆解 → 输出（中英双语）→ 自检。

详见 [QUICKSTART.md](QUICKSTART.md)。

## 5 阶段流水线

```
阶段 0 · 确定骨架  →  阶段 1 · 变量填充  →  阶段 2 · 锚点拆解  →  阶段 3 · 输出  →  阶段 4 · 自检
```

## 设计原则

| # | 原则 | 说明 |
|---|---|---|
| ① | **骨架变量分离** | 结构一次定义，内容每次填充，换变量不换骨架 |
| ② | **变量可追溯** | 每个值可回溯到用户输入或推导逻辑 |
| ③ | **风格可执行** | 不依赖「高级」「好看」等主观词，7 维锚点可被 AI 直接执行 |
| ④ | **框架不绑定内容** | 图像生成 / 文章写作 / 视频脚本均适用同一套机制 |
