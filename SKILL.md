# 通用提示词框架生成器

**Prompt 的乐高系统。**

骨架是底板，变量是积木块——底板的孔位定了，积木的颜色、形状、排列可以自由替换。换一套积木就是一件全新的作品，但结构永远工整、稳定、可复用。

## 核心机制

```
骨架（不变）  +  变量（可变）  =  最终提示词
   ↑                ↑
 一次定义       每次填入
```

骨架是模具，变量是材料。换个变量，同一个骨架产出完全不同但结构一致的提示词。

## 工作流程

### 阶段 0：确定骨架结构

在填充变量之前，先和用户确认本次任务需要什么骨架。

**两种模式：**

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **A. 使用预设骨架** | 从[骨架库](references/skeleton-library.md)中选择一个现成骨架 | 电商封面 / 海报 / 文章配图等常见任务 |
| **B. 自定义骨架** | 根据用户描述，现场搭建一个骨架 | 特殊创意需求，骨架库里没有匹配的 |

**自定义骨架的搭建规则：**

骨架由 3 部分组成，每个部分包含若干插槽：

```
┌─────────────────────────────────┐
│ ① 角色与任务层                   │
│   角色定义 - 你是谁              │
│   任务定义 - 要做什么            │
│   格式约束 - 比例/尺寸/时长       │
├─────────────────────────────────┤
│ ② 风格与约束层                   │
│   风格锁定 - 视觉/语言规则       │
│   正向约束 - 必须包含            │
│   负向约束 - 严格避免            │
├─────────────────────────────────┤
│ ③ 内容与细节层                   │
│   主体 - 画面核心               │
│   动作/状态 - 主体在做什么       │
│   场景/环境 - 在哪              │
│   道具/元素 - 辅助物品           │
│   文案/文字 - 画面上的字         │
│   点缀/氛围 - 细节质感           │
└─────────────────────────────────┘
```

每个 `{插槽}` 就是一个变量位。骨架中插槽的数量和名称由内容类型决定。

### 阶段 1：填充变量

确定骨架后，填充全部变量。变量分两级：

**核心变量（必问用户）：**

决定输出方向的关键变量。用户必须确认，Claude 不替用户决定：

- 内容主体是什么
- 核心风格/调性
- 关键约束（如必须出现/必须不含的元素）
- 文字/文案（如有）

**衍生变量（Claude 自动推导）：**

可由核心变量 + 常识推导的变量，Claude 自行填充后告知用户：

- 尺寸/比例（根据使用场景推断）
- 具体构图/视角（根据风格推导）
- 色彩/色调（根据调性推导）
- 边缘细节（背景/点缀/氛围）

**变量填充规则：**
- 每个变量用中文填写，具体且可视化，避免抽象形容词
- 每个变量只描述一个维度，不混合多个信息
- 如果某个变量不适用当前任务，填「无」而非删除该变量位
- 变量值必须可被目标 AI 模型直接理解，不用行业黑话

### 阶段 2：风格锚点拆解（核心创新）

对于任何涉及视觉/风格的任务，将模糊的风格描述拆解为 **7 维锚点**：

```
视角 → 前景 → 主体 → 构图/排版 → 元素风格 → 色调 → 氛围
```

每个维度一句话，遵循三条铁律：
1. **≤ 30 字**，不展开
2. **零抽象形容词**，如用「高光偏青，阴影偏暖」代替「日系清新」
3. **AI 可直接执行**，用摄影/绘画/设计通用词汇

> 非视觉任务可替换为对应的风格维度，但拆解粒度原则不变。

### 阶段 3：输出最终提示词

将填充后的变量代入骨架，输出完整提示词。

**输出格式：**

```
## 🎯 最终提示词

[代入变量后的完整版，中英双语（如适用）]

## 📋 变量回执

| # | 变量 | 类型 | 最终值 |
|---|------|------|--------|
| 1 | ... | 核心 | ... |
| 2 | ... | 衍生 | ... |

## 🔧 使用建议

- 推荐模型：[如有]
- 建议参数：[如有]
```

### 阶段 4：自检

输出前静默检查：

- [ ] 所有变量已填充，无 `{ }` 残留
- [ ] 核心变量均来自用户输入，未擅自替换
- [ ] 风格锚点覆盖 ≥ 5 个维度
- [ ] 每条锚点 ≤ 30 字，无抽象形容词
- [ ] 画面/内容描述具体、可视化、可被目标模型直接执行
- [ ] 正负向约束不矛盾

## 设计原则

1. **骨架和变量分离** — 骨架定义结构（一次定义），变量提供内容（每次填充），换变量不换骨架
2. **变量可追溯** — 每个最终值都能回溯到用户输入或推导逻辑，不做黑箱替换
3. **风格可执行** — 不依赖「高级」「好看」等主观词，每个描述都是 AI 可执行的具体指令
4. **框架不绑定内容** — 这套机制适用于图像生成、文章写作、视频脚本等任何需要结构化提示词的场景
